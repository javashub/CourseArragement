## Context

当前管理员工作台位于 [`UI/coursearrange/src/views/dashboard/DashboardView.vue`](/Users/lyk/dev/project_source/毕业设计/CourseArrange/UI/coursearrange/src/views/dashboard/DashboardView.vue)，已经接入登录用户、角色、权限等真实上下文，但页面仍偏“开发联调展示”，没有成为管理员日常进入系统后的主控台。与此同时，左侧菜单已经补齐了 `权限管理` 入口，因此首页中的“权限管理联调入口”卡片不再承担必要导航职责，反而会打断信息层次。

这个页面已经建立了学院档案馆式的视觉基调，后续优化应延续这条路线，而不是切回常规后台卡片堆叠。约束是只做前端单页改造，不改动后端接口和数据库菜单结构，并保持 Vue 3 + Element Plus 现有工程模式可维护。

## Goals / Non-Goals

**Goals:**
- 将管理员工作台重新定义为“排课系统主控台”，突出当前账号、系统主链状态、排课域状态和系统治理进度。
- 删除首页中的“权限管理联调入口”卡片，避免与左侧菜单重复。
- 保留现有学院档案馆式风格，但强化排版、信息模块、节奏和视觉记忆点。
- 保证桌面端和移动端都能稳定展示，不引入额外依赖。

**Non-Goals:**
- 不新增后端接口或实时统计接口。
- 不改动左侧菜单、RBAC 逻辑或路由结构。
- 不把教师端、学生端页面并入管理员工作台。
- 不做整站主题重构，只优化工作台页面本身。

## Decisions

1. 采用“主视觉横幅 + 关键指标条 + 状态卡组 + 当前阶段清单”的信息架构。
原因：当前页面已有用户上下文和简要卡片，但缺少首页应有的主次关系。新的结构可以让管理员先看到身份和系统状态，再看到重点模块和阶段任务。
备选方案：继续沿用纯卡片网格。未采用，因为会让首页继续像中间页而不是入口页。

2. 删除“权限管理联调入口”卡片，用更通用的“系统主链状态”或“当前阶段重点”模块替代。
原因：权限管理已经在左侧菜单中稳定存在，首页不需要重复提供显式按钮。首页应服务总览，而不是单点跳转。
备选方案：保留卡片但弱化样式。未采用，因为仍然会占据视觉权重。

3. 视觉上延续“学院档案馆”方向，但加入更鲜明的编排元素，例如编号、档案签条、状态标签、分栏模块和更明确的层级对比。
原因：这能保持已有风格连续性，同时让页面从“好看”进化为“有控制台感”。这也符合 `frontend-design` 要求中的强风格执行，而不是通用后台模板。
备选方案：改成极简数据看板。未采用，因为会损失当前项目已经形成的辨识度。

4. 数据内容优先使用现有前端可直接获得的信息和静态说明文案。
原因：当前没有为首页单独准备统计 API，强行拼装假动态数据会让设计失真。先用真实上下文和项目阶段信息构成高质量首页，更稳妥。
备选方案：新增接口或复用多个模块接口聚合。未采用，因为超出本次变更范围。

## Risks / Trade-offs

- [页面过度装饰导致信息可读性下降] → 保持模块数量可控，强调标题、数字和摘要的主次关系，避免纯视觉堆叠。
- [没有新增后端数据导致首页“统计感”不足] → 使用“状态总览 + 阶段进度 + 主链说明”的表达方式替代空洞数字。
- [移动端布局被复杂版式压垮] → 设计上优先使用可折叠为单列的网格和分组，不依赖绝对定位完成核心信息排布。
- [删除权限管理卡片后用户误以为入口消失] → 在首页其它模块中弱化提及“系统管理已迁入左侧菜单”，但不再做显式 CTA 卡片。
