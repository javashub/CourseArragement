## 当前主要缺口

### 一、需求对齐缺口

- 时间片模板、连堂上限、教师日上限、教师禁排、班级禁排和任务优先级已经接入排课入口，但教师连排偏好、班级连续上课限制等更细粒度的软约束规则还没有形成完整的可配置闭环。
- 管理员首页统计项还未完全补齐到需求文档列出的范围。
- 教师端、学生端仍有部分能力停留在旧接口或简化版页面，和需求文件相比不完整。

### 二、后端结构缺口

- `ClassTaskServiceImpl`、`ClassUtil` 仍然偏大，排课算法内部仍保留历史基因编码和过渡对象语义。
- 旧 JWT 与兼容登录链路尚未彻底移除，密码改造仍处于“BCrypt + 兼容迁移”状态，不是彻底收尾状态。
- 包结构仍是新旧并存，`controller / service / entity / request / response / resource / organization / system / schedule` 没有完全收成统一分层。
- OpenAPI 文档、数据库迁移脚本、统一部署文档仍未补齐。

### 三、前端质量缺口

- 旧页面虽然大部分已切到统一 API，但页面拆分和通用组件抽离还不够，`ClassTaskList.vue` 等页面仍偏大。
- 管理员核心菜单虽然已经补齐到权限上下文，但教师端、学生端菜单和权限配置页体验仍需继续打磨。
- 还没有前端单元测试、组件测试、E2E 测试。
- 交互层面仍缺少更完整的空状态、异常状态、重试指引和一致的表单交互模式。
